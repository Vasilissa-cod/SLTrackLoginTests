# SLTrackLoginTests

Проверка входа в систему, отображения приветствия и фильтрации задач по пользователю

## Описание задачи

Автоматизированные тесты для проверки функциональности системы управления задачами SL-Track.

## Шаги тестирования

### 1. Открыть страницу https://slqamsk.github.io/cases/sltrack/v02

**Ожидаемый результат:**
- Заголовок страницы: SL-Track — Система управления задачами
- Лейблы полей ввода: Email:, Пароль:
- Плейсхолдеры полей ввода: user@example.com, ••••••••

### 2. Ввести admin@example.com в поле Email

### 3. Ввести admin123 в поле Пароль

### 4. Нажать кнопку Войти

**Ожидаемый результат:**
- Приветствие на странице: присутствует
- Имя пользователя в приветствии отображается корректно: Пётр Иванов
- Роль пользователя отображается корректно: Администратор
- Есть кнопка Выйти

### 5. (*) Нажать кнопку "Мои задачи"

**Ожидаемый результат:**
- Отображаются только задачи пользователя Пётр Иванов

### 6. (*) Анализ результата выполнения теста для страницы v03

Проверка работы теста на версии v03 системы.

## Технологический стек

- **Java 11+** - язык программирования
- **Selenide** - фреймворк для автоматизации веб-тестирования
- **JUnit 5** - фреймворк для unit-тестирования
- **Allure Report** - генерация отчетов о тестировании
- **Gradle** - система сборки проекта

## Структура тестов

### testLoginAndTaskFiltering()
Основной тест для проверки входа в систему, отображения приветствия и фильтрации задач (v02).

### testLoginAndTaskFilteringV02()
Тест на основе примера кода с использованием селекторов By.id и By.className (v02).

### testAnalyzeV03Page()
Анализ результата выполнения теста для страницы v03.

### testAnalyzeV03PageDetailed()
Детальный анализ страницы v03 с проверкой всех элементов.

## Запуск тестов

```bash
# Запуск всех тестов
./gradlew test

# Запуск конкретного класса тестов
./gradlew test --tests SLTrackLoginTests
```
